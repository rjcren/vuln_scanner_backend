PHP open_basedir Is Not Configured low The open_basedir configuration directive will limit the files that can be opened by PHP to the specified directory-tree. When a script tries to open a file with, for example, fopen() or gzopen(), the location of the file is checked. For a remote attacker it is not possible to break out of the open_basedir restrictions if he is only able to inject the name of a file to be included.
